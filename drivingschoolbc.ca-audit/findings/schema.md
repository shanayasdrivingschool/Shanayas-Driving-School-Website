# Schema.org / Structured Data Audit — drivingschoolbc.ca (Shanaya's Driving School)

Audited: 2026-07-02
Scope: all 38 pre-rendered routes in `public_html/` (generated by `scripts/generate-static-seo-pages.mjs`) plus the React runtime source (`src/components/SeoManager.tsx`) that re-injects the same JSON-LD client-side on route change.

## 1. How schema is delivered (architecture)

Two code paths produce **identical** JSON-LD, which is good for consistency:

1. **Build-time (static HTML)** — `scripts/generate-static-seo-pages.mjs` runs as an npm `postbuild` step. It clones `public_html/index.html` as a template and stamps `<script id="local-business-schema">` and `<script id="faq-schema">` into every route's `index.html`. This is what crawlers see with zero JS execution — correct approach for a Vite/React SPA.
2. **Runtime (client-side)** — `src/components/SeoManager.tsx` runs on every route change and calls `setJsonLd()` to overwrite the same two `<script>` tags (`local-business-schema`, `faq-schema`) using `document.head.querySelector`. Confirmed the object literals in both files are byte-identical (same `@id`, address, areaServed, sameAs).

No other JSON-LD generators exist in `src/` — confirmed via repo-wide search for `ld+json`, `LocalBusiness`, `FAQPage`, `geo`, `openingHours`, `aggregateRating`, `hasOfferCatalog`, `BreadcrumbList` (only the shadcn/ui presentational `<Breadcrumb>` component and a visual (non-schema) breadcrumb trail in `src/pages/BlogPost.tsx` — no `BreadcrumbList` JSON-LD anywhere).

## 4. Real-data check before recommending Review/AggregateRating

`src/components/TestimonialsSection.tsx` renders 8 hardcoded testimonials (`defaultReviews`) — quote, first-name + last-initial, and a city (e.g. "Student - Langford"). Checked for:

- Star rating value per review: **not present** — the UI renders a static `*****` text glyph identically for every card, not a data-driven rating field.
- Timestamp/date per review: **absent**.
- Source/platform attribution (Google, Facebook, etc.): **absent** — no link to an external review platform anywhere in the codebase, and no Google Business Profile URL in `sameAs`.
- Review count: **absent** (8 items is a fixed, hand-authored array, not a live feed).

**Conclusion: this is marketing copy, not structured review data.** Per instructions, no `Review`/`AggregateRating` JSON-LD has been generated. Flagging as needs-real-data (see Finding S-06) — recommend wiring this section to a verified source (Google Business Profile API/embed, or a review platform such as Trustpilot/Facebook Recommendations) before adding `AggregateRating`. Adding fabricated ratings risks a manual action under Google's structured data guidelines.
